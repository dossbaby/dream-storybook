# 점AI 콘텐츠 커뮤니티 전략 가이드

---

## 점AI의 본질

### 한 문장 정의

**"고민을 던지고, 답을 찾고, 그 결과가 어땠는지 나누는 곳"**

### 핵심 구조: 질문 = 게시글

```
┌─────────────────────────────────────────────────────────────┐
│  [질문 스레드]                                               │
│                                                             │
│  "3년 사귄 남자친구가 결혼 얘기를 피해요. 기다려야 할까요?"    │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  👤 질문자 피드백:                                           │
│  "타로 보고 용기 내서 직접 물어봤어요.                        │
│   알고 보니 집 문제로 고민 중이었대요. 카드 맞았음 ㅠㅠ"       │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  💬 같은 고민으로 본 사람들 (89명)                            │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 🔮 기다리는토끼: "저도 비슷한 상황이었는데 기다렸더니     │   │
│  │    결국 헤어졌어요. 미리 얘기할걸..."                    │   │
│  │    [내 리딩 보기 🔓]                                    │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 🔮 고민하는코끼리: "똑같은 상황이라 봤는데 긍정 카드      │   │
│  │    나왔어요. 희망을 가져봅니다"                          │   │
│  │    [리딩 비공개 🔒]                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  [🔮 나도 이 고민으로 타로 보기]                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 왜 이 구조인가

**기존 문제:**
- 리딩 결과 공개 = 부담스러움
- 남의 카드 결과 = 별로 안 궁금함
- 그냥 결과만 보여주면 = "그래서 뭐?"

**새 구조의 해결:**
- 질문만 공개 = 부담 없음
- 리딩 결과는 선택적 공개 = 본인이 컨트롤
- 핵심 콘텐츠는 "그래서 어떻게 됐는데?" = 이게 진짜 궁금한 것
- 같은 고민 참여 = 내 피드백이 댓글이 됨

### 사람들이 진짜 궁금한 것

```
❌ "어떤 카드가 나왔는지"
✅ "그래서 어떻게 됐는지"
```

"이직 타로 봤더니 탑 카드 나왔어요" → 별로 안 궁금함
"이직 타로 보고 옮겼는데 진짜 잘된 선택이었어요" → 궁금함

카드 결과보다 **결과 검증**이 핵심 콘텐츠다.

### 공개 설정 구조

| 요소 | 기본값 | 변경 가능 |
|------|--------|-----------|
| 질문 내용 | 공개 | 비공개 가능 (피드 안 올라감) |
| 리딩 결과 (카드/해석) | 비공개 | 공개 가능 |
| 내 피드백 | 공개 | 비공개 가능 |

**핵심**: 질문과 피드백만 공개해도 콘텐츠가 된다. 리딩 결과 공개는 보너스.

### 핵심 경험 루프

```
고민이 있다 → 질문 스레드 생성 → 리딩 받음 → 
시간이 지남 → 결과 피드백 작성 ("어떻게 됐는지") →
다른 사람들이 이 스레드를 봄 →
"나도 비슷한 고민" → 같은 질문으로 리딩 →
내 피드백이 댓글로 달림 → 스레드가 풍성해짐 → 반복
```

### 경쟁 서비스와의 차이

| 기존 타로 앱 | 커뮤니티 | 점AI |
|-------------|----------|------|
| 혼자 결과 받고 끝 | 글 쓰고 댓글 받음 | 리딩이 참여 방식 |
| 결과 공개 부담 | 익명이어도 글쓰기 부담 | 질문만 공개, 부담 없음 |
| 맞았는지 모름 | 검증 불가 | 피드백으로 검증 |
| 1회성 | 글 올리면 끝 | 스레드가 계속 성장 |

### 성공 지표

점AI가 제대로 작동하면:
- 리딩 후 평가 응답률 70% 이상 (👍👎 클릭)
- 피드 스레드 클릭률 5% 이상
- "나도 이 고민으로" 전환율 20% 이상
- 스레드당 평균 참여자 5명 이상
- 👍 긍정 평가 비율이 전체 신뢰도 지표가 됨

## 핵심 인사이트

사람들은 "타로 결과"에 관심이 없다. 사람들은 "나와 비슷한 상황에 있는 사람의 이야기"에 관심이 있다. 블라인드가 성공한 이유는 익명의 누군가가 올린 고민에 자신을 대입하고, 의견을 나누고, "나만 이런 게 아니구나"라는 연결감을 느끼기 때문이다. 점AI의 공유 콘텐츠도 같은 원리로 작동해야 한다.

## 1. 사용자 입력 유도 전략

### 문제
사용자가 "연애운 봐줘"처럼 짧게 입력하면, 아무리 좋은 제목 프롬프트를 써도 뽑아낼 스토리가 없다.

### 해결: 입력 단계에서 구체성 유도

**프롬프트 박스 플레이스홀더 예시들 (랜덤 로테이션)**
```
"3개월 준비한 면접 결과를 기다리고 있는데, 붙었을까요?"
"남자친구가 요즘 바쁘다며 연락이 뜸한데, 진짜 바쁜 걸까요?"
"이직 제안을 받았는데 지금 회사에 남을지 옮길지 모르겠어요"
"짝사랑하는 사람에게 고백해도 될까요?"
"창업을 준비 중인데 이 아이템으로 성공할 수 있을까요?"
"결혼을 앞두고 있는데 이 사람이 맞는 사람일까요?"
```

**입력창 위 가이드 문구**
```
💡 상황을 구체적으로 적을수록 더 정확한 리딩을 받을 수 있어요
```

**입력 최소 글자 수**: 20자 이상 권장 (강제는 X, 부드러운 넛지)
```
"조금 더 자세히 적어주시면 더 깊은 리딩을 해드릴 수 있어요"
```

## 2. 제목 생성 프롬프트

### 원칙
- 결과가 아닌 상황을 강조
- 감정적 연결고리 생성
- 궁금증 유발 (클릭베이트 아닌 진짜 궁금증)
- 나도 비슷한 상황이라는 공감 트리거

### 제목 생성 시스템 프롬프트

```
당신은 점AI의 제목 생성 전문가입니다. 사용자의 타로 질문과 리딩 결과를 바탕으로, 다른 사람들이 클릭하고 싶어지는 제목을 만듭니다.

## 제목 작성 원칙

1. **상황 중심**: 타로 결과가 아니라 질문자의 상황을 부각
   - ❌ "연인 카드가 나온 타로 리딩"
   - ✅ "3년 사귄 남친이 갑자기 결혼 얘기를 피하기 시작했다"

2. **감정 연결**: 누구나 공감할 수 있는 감정 키워드 포함
   - 불안, 기대, 혼란, 희망, 두려움, 설렘, 답답함

3. **구체적 디테일**: 추상적 표현 대신 구체적 상황
   - ❌ "이직 고민"
   - ✅ "연봉 1500 올려준다는데 지금 회사 동료들이 너무 좋다"

4. **열린 결말**: 답을 주지 말고 궁금증 유발
   - ❌ "헤어지는 게 맞다고 나온 타로"
   - ✅ "계속 만나도 될까? 6개월째 답 없는 썸남"

5. **1인칭 또는 3인칭 상황 묘사**
   - "면접 5번 떨어지고 6번째 도전하는 중"
   - "결혼 3년차, 남편이 요즘 이상하다"

## 제목 포맷

- 길이: 25-45자
- 이모지: 사용하지 않음
- 말투: 구어체, 일기/고민글 느낌
- 마침표: 생략 가능

## 입력 정보

```

## 3. 피드 썸네일 이미지 프롬프트

### 원칙
- 얼굴이 안 보이는 익명성 (블라인드처럼)
- 상황을 시각적으로 암시
- 감정적 톤 전달
- 너무 직접적이지 않은 은유적 표현
- 클릭 유도하는 시각적 호기심

### 이미지 생성 시스템 프롬프트

```
당신은 점AI의 썸네일 이미지 프롬프트 생성 전문가입니다. 사용자의 고민 상황을 시각적으로 표현하는 이미지 프롬프트를 만듭니다.

## 이미지 스타일 가이드

1. **익명성 유지**
   - 인물이 나올 경우: 뒷모습, 실루엣, 흐릿한 윤곽
   - 얼굴이 보이지 않는 구도
   - 또는 인물 없이 상황을 암시하는 오브젝트만

2. **감정적 분위기**
   - 색감으로 감정 전달 (따뜻함/차가움/불안/희망)
   - 조명으로 무드 설정 (새벽/황혼/밤/빛 줄기)
   - 날씨로 심리 표현 (비/안개/맑음/구름)

3. **상황 암시 오브젝트**
   - 연애: 두 개의 컵, 시든 꽃, 폰 화면, 빈 의자
   - 직장: 창밖 보는 뒷모습, 두 갈래 길, 닫힌 문
   - 시험: 책상 위 책들, 새벽 창문, 커피잔
   - 결정: 갈림길, 열쇠, 닫힌 상자

4. **아트 스타일**
   - 몽환적이고 부드러운 일러스트
   - 미니멀하면서도 감정적
   - 과하지 않은 색상 (2-3색 중심)
   - 동양적 감성 + 현대적 미니멀

## 금지 요소
- 명확한 얼굴
- 타로 카드 직접 노출
- 텍스트 삽입
- 과도하게 어둡거나 무서운 분위기
- 너무 구체적인 상황 묘사 (상상의 여지 필요)

## 입력 정보
- 사용자 질문: {user_question}
- 핵심 감정: {detected_emotion}
- 상황 카테고리: {category}

## 출력
DALL-E / Midjourney 호환 이미지 프롬프트 (영문)
```

### 4-3. 피드백 시스템

#### 핵심: 리딩 직후 바로 평가

**한 달 뒤 후기는 비현실적.** 앱 삭제, 알림 끔, 귀찮음. 리텐션 기대하기 어려움.

**해결: 결과 나오자마자 물어보기**

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  ✨ 리딩이 완료됐어요                                        │
│                                                             │
│  [리딩 결과 내용...]                                        │
│                                                             │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│                                                             │
│  이 해석, 와닿나요?                                         │
│                                                             │
│  [👍 와닿아요]    [👎 별로예요]                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 스팀 스타일 심플 평가

**두 개만. 끝.**

| 평가 | 의미 | 집계 |
|------|------|------|
| 👍 | 이 해석이 내 상황에 맞는 것 같다 | 긍정 평가 |
| 👎 | 잘 모르겠다 / 안 맞는 것 같다 | 부정 평가 |

**표시 방식 (스팀처럼)**

```
👍 긍정적 (87%)  — 1,247명 평가
```

```
👍 매우 긍정적 (94%)  — 3,891명 평가
```

```
👍👎 복합적 (62%)  — 523명 평가
```

#### 이 방식의 장점

1. **즉시 수집**: 리딩 직후라 응답률 높음
2. **부담 없음**: 클릭 한 번이면 끝
3. **바로 쌓임**: 한 달 기다릴 필요 없음
4. **해석 품질 지표**: "이 카드 조합 + 이 질문 유형 = 긍정 평가 높음" 데이터

#### 평가 데이터 활용

**스레드 내 표시**
```
📊 이 고민 통계
• 47명 참여
• 👍 긍정적 (78%)
• 가장 많이 나온 카드: 연인
```

**카드 조합별 통계**
```
🎴 '탑 + 별 + 태양' 조합

이 조합을 받은 892명 중:
👍 긍정적 (84%)

이 조합이 자주 나온 고민:
• 이직/퇴사 (34%)
• 큰 변화 앞둔 결정 (28%)
```

#### 추가 피드백 (선택적)

평가 후 부드럽게 유도:
```
👍 선택함

한마디 남기실래요? (선택)
[                                        ]

💡 같은 고민 가진 분들에게 도움이 돼요

[건너뛰기]  [등록하기]
```

**한마디는 보너스.** 없어도 스레드가 돌아감. 있으면 더 풍성해지는 것.

#### 재방문 시 후속 피드백

사용자가 나중에 다시 왔을 때 (새 리딩 받으러 왔을 때):

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  📬 지난번 리딩, 어떻게 됐어요?                              │
│                                                             │
│  "남자친구가 연락이 뜸해져서..."                             │
│  2주 전에 봤던 리딩                                         │
│                                                             │
│  [👍 맞았어요]  [👎 안 맞았어요]  [⏭️ 나중에]               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**푸시로 끌어오는 게 아니라, 알아서 온 사람한테 물어보기.**
어차피 점 보러 다시 올 사람은 또 와. 그때 잡으면 됨.

#### Verified Reader란?

**아마존 "Verified Purchase" 개념 차용**

| 배지 | 의미 | 신뢰도 |
|------|------|--------|
| 🔮 Verified Reader | 같은 질문으로 리딩 받음 | 높음 |
| 🔮 + 👍 | 리딩 받고 + "와닿아요" 평가 | 높음 |
| 🔮 + 👎 | 리딩 받고 + "별로예요" 평가 | 참고용 |
| 👤 일반 | 리딩 없이 댓글만 | 보통 |

**왜 이게 중요한가:**
- Verified 댓글은 "이 사람도 진짜 이 고민을 했구나" 신뢰감
- 적중 배지까지 있으면 "실제 검증된 경험담" 무게감
- 일반 댓글도 허용해서 참여 허들은 낮게 유지

#### 두 가지 참여 경로

```
[경로 1: 가벼운 참여]
스레드 봄 → 공감됨 → 댓글 쓰기 → 텍스트만 작성 → 완료
                                    (👤 일반 댓글)

[경로 2: 깊은 참여]  
스레드 봄 → 공감됨 → "나도 이 질문으로 타로 보기" →
바로 카드 선택 화면 → 카드 뽑음 → 리딩 완료 →
한마디 작성 (선택) → 완료
                    (🔮 Verified Reader 댓글)
```

```
[공감 텍스트 스트리밍 + "알림 받고 둘러보기" 버튼 표시]
→ 버튼 클릭 → 피드로 이동 (백그라운드 분석 계속)
→ 완료 시 푸시/인앱 알림 → 내 리딩 페이지로 이동
```

**UI 문구**
```
🔔 분석 완료되면 알려드릴게요
   그동안 다른 사람들의 리딩 둘러보기
```

**선순환 구조**
```
내 리딩 대기 → 피드 둘러봄 → 공감되는 리딩 발견 →
"나도 같은 고민" 클릭 → 새 리딩 시작 (대기) →
또 피드 둘러봄 → 원래 리딩 완료 알림 → 확인 →
새 리딩도 곧 완료 → 두 개 리딩 결과 비교
```

한 세션에서 여러 리딩이 동시에 돌아가는 구조 = 체류 시간 ↑, 이탈률 ↓

**알림 유형**
- 인앱 알림: 앱 내에 있을 때 상단 토스트
- 푸시 알림: 앱 나갔을 때 (허용한 경우)
- 알림 문구: "✨ 타로 리딩이 완료됐어요! 카드가 전하는 메시지를 확인하세요"

**대기 중 상태 표시**
피드 상단에 작은 프로그레스 바 또는 뱃지
```
[🔮 내 리딩 분석 중... 약 1분 남음]
```
클릭하면 바로 해당 리딩 페이지로

## 11. 수익화 전략

### 핵심 원칙: 커뮤니티 먼저, 수익은 나중에

**왜?**

점AI의 핵심 가치는 "더 좋은 AI 해석"이 아니라 "다른 사람들은 어떻게 됐는지"야. 커뮤니티 경험이 핵심인데, 초기에 유료 벽 세우면:

- 리딩 수 감소 → 콘텐츠 안 쌓임 → 커뮤니티 안 돌아감
- "여기 볼 게 없네" → 이탈 → 악순환

**임계점 먼저 넘기기**

스레드가 풍성해지고, 사람들이 "여기 오면 볼 게 많다"고 느끼는 상태가 먼저. 그 다음에 수익화.

### 하지 말아야 할 것: 기능 차별화

```
❌ 무료: GPT-3.5 / 유료: GPT-4
❌ 무료: 짧은 해석 / 유료: 긴 해석
❌ 무료: 기본 카드 / 유료: 프리미엄 카드
```

**왜 안 되는가:**
- 핵심 경험(리딩)의 품질 차이 → 무료 사용자 불만
- 무료 사용자도 콘텐츠 생산자인데, 품질 낮은 콘텐츠가 쌓임
- 커뮤니티 가치와 무관한 차별화

### 해야 할 것: 편의성 차별화

```
✅ 무료: 핵심 기능 전부 (리딩, 스레드 참여, 평가)
✅ 유료: 더 편하게, 더 깊게
```

### 유료 플랜 설계 (안)

#### 무료 (기본)

- 리딩 무제한 (단, 동시 분석 1개)
- 스레드 생성 및 참여
- 👍👎 평가 및 한마디
- 피드 탐색
- 기본 통계 보기

#### 프리미엄 (월 구독)

**편의성 강화:**
- 동시 분석 3개 (기다리면서 여러 개 돌리기)
- 광고 없음
- 새 댓글/참여자 실시간 알림

**깊은 탐색:**
- 내 카드 패턴 분석 ("당신에게 자주 나오는 카드")
- 내 고민 히스토리 통계
- 비슷한 고민 적중 후기 먼저 보기
- 카드 조합별 상세 통계

**커뮤니티 특권:**
- Verified Reader 댓글 상단 노출
- 프리미엄 배지 (선택적 표시)

### 대안 수익 모델

#### 1. 광고 (초기)
- 피드 사이사이 네이티브 광고
- 리딩 대기 중 광고 (알림 받고 둘러보기 선택 시)

#### 2. 특별 리딩 상품 (일회성)
- 연말 특별 리딩 (유료)
- 신년 운세 상세 분석 (유료)
- 궁합 리딩 (두 사람 동시 분석)

#### 3. 후원/팁
- 마음에 드는 리딩 후 "커피 한 잔 사주기" 버튼
- 부담 없는 소액 결제

### 수익화 타이밍

```
[Phase 1: 커뮤니티 구축] — 0-3개월
- 무료로 풀어서 콘텐츠 축적
- 스레드 1,000개 이상 목표
- DAU 안정화

[Phase 2: 광고 도입] — 3-6개월  
- 네이티브 광고 시작
- 사용자 반응 모니터링

[Phase 3: 프리미엄 출시] — 6개월+
- 유료 구독 옵션 추가
- 특별 리딩 상품 출시
```

### 핵심 메트릭

| 지표 | Phase 1 목표 | 수익화 시점 목표 |
|------|-------------|-----------------|
| 스레드 수 | 1,000+ | 10,000+ |
| DAU | 100+ | 1,000+ |
| 스레드당 참여자 | 3+ | 5+ |
| 재방문율 (7일) | 20%+ | 40%+ |

**수익보다 지표 먼저.** 지표가 안정되면 수익은 따라온다.

## 부록: 아이디에이션 (미래 검토용)

> ⚠️ 아직 확정 안 된 아이디어들. MVP 돌아가고 데이터 쌓이면 검토.

### 레딧 스타일 피드 구조

```
레딧                    점AI
─────────────────────────────────────
서브레딧               카테고리 (연애/직장/학업)
포스트                 질문 (스레드)
업보트                 "나도 타로 보기" (참여)
댓글                   리딩 후 한마디 (Verified 댓글)
카르마                 활동 점수 (리딩 수 + 댓글 수)
Hot/Rising            핫한 고민 (참여자 급증)
```

### 랭킹 시스템

**유저 랭킹 요소 (카르마 같은 것)**
- 내가 올린 질문에 참여자 수
- 내가 단 댓글에 공감 수
- 리딩 횟수
- 👍 받은 비율

**스레드 랭킹 (Hot 알고리즘)**
- 참여자 수 (나도 타로 보기 클릭)
- 시간당 참여 증가율
- 댓글 수
- 평균 만족도 (👍 비율)

### 피드 레이아웃 옵션

**옵션 A: 단일 피드 (현재)**
```
[메인 피드]
- 질문들 나열
- 정렬: 핫 / 최신 / 카테고리
```

**옵션 B: 레딧 스타일 (2컬럼)**
```
[왼쪽: 메인 피드]     [오른쪽: 사이드바]
- 질문들              - 🔥 핫한 고민 TOP 5
                      - 📊 오늘의 통계
                      - 🏆 활발한 유저
```

**옵션 C: 트위터 스타일**
```
[For You] [Following] [연애] [직장] [학업]
─────────────────────────────────────────
개인화 피드 / 팔로우한 태그 / 카테고리별
```

### 참여 플로우 확장 아이디어

**"나도 타로 보기" 했을 때:**
1. 스레드 참여자 수 +1
2. 내 리딩 끝나면 한마디 작성 (선택)
3. 한마디 = 그 스레드의 댓글로 등록
4. 내가 👍 누르면 스레드 만족도에도 반영

**Verified 댓글 레벨 분화**
```
🔮 Verified Reader — 리딩만 받음
🔮👍 — 리딩 + 만족 평가
🔮💬 — 리딩 + 한마디까지 작성
```

### 게이미피케이션 아이디어

**배지 시스템**
- 🌱 첫 리딩
- 🔥 10회 리딩
- 💬 첫 댓글
- ⭐ 공감 100개 받음
- 🎯 만족도 90% 이상 유지

**레벨 시스템**
```
Lv.1 호기심 (0-5 리딩)
Lv.2 탐색자 (6-20 리딩)  
Lv.3 구도자 (21-50 리딩)
Lv.4 현자 (51+ 리딩)
```

### 소셜 기능 아이디어

- 비슷한 고민 패턴 유저끼리 매칭?
- 익명 DM?
